# 💻 Lógica de Programação e Algoritmos: Notas de Aula

Este documento serve como guia central para as aulas de lógica, cobrindo desde a estruturação do pensamento computacional até a modelagem e documentação de algoritmos.

## 📅 1. Conteúdo de Aula

### Introdução à Lógica e Algoritmos
* **Conceito:** Sequência de passos finitos e não ambíguos para resolver um problema.
* **Formas de Representação:** Descrição narrativa, pseudocódigo (Portugol) e fluxogramas.
* **Estruturas Fundamentais:**
  * Variáveis, constantes e tipos de dados (Inteiro, Real, Caractere, Booleano).
  * Operadores aritméticos, relacionais e lógicos.
  * Estruturas condicionais (`SE-ENTÃO-SENÃO`, `ESCOLHA-CASO`).
  * Estruturas de repetição (`ENQUANTO`, `FAÇA-ENQUANTO`, `PARA`).

## 📋 2. Requisitos Aplicados ao Software (Funcionais e Não Funcionais)

*Mesmo na lógica básica, o aluno deve entender o que o algoritmo precisa entregar e sob quais condições.*

### Requisitos Funcionais (RF)
*O que o algoritmo deve executar computacionalmente.*
* **RF01:** O algoritmo deve receber quatro notas bimestrais do aluno.
* **RF02:** O sistema deve calcular a média aritmética simples dessas notas.
* **RF03:** O programa deve exibir "Aprovado" se a média for igual ou maior que 7.0.

### Requisitos Não Funcionais (RNF)
*Restrições técnicas ou operacionais do código.*
* **RNF01 (Linguagem):** O código deve ser implementado estritamente em linguagem C (ou Python).
* **RNF02 (Ambiente):** O programa deve rodar via console/terminal de texto.
* **RNF03 (Estilo):** O código-fonte deve ser indentado e conter comentários explicando as funções.
